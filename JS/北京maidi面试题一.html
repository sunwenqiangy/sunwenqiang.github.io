# 一、简答题

# 1.说出三种不同含义this的使用场景。

①在全局环境中，或作普通函数被调用，this 指向 Window

②使用 call apply bind 改变this指向，传入什么绑定什么

③如果函数作为对象的方法时，方法中的 this 指向该对象

④在class 的方法中调用或作为构造函数调用 this 指向当前实例本身

⑤箭头函数 this指向上级作用域中this的值

# 2.实现一个获取当前页面链接search值的函数。（如下例子中，输入key1输出value1）

例：’http://test.com/index?key1=value1&key2=value2’

function getSearchValue() { 
    var arr = window.location.search.substring(1).split('&');
    var obj = {};
    var len = arr.length;
    for (var i = 0; i < len; i++) {
        var key = arr[i].split('=')[0];
        var value = arr[i].split('=')[1];
        obj[key] = value;   
    }
    var k = prompt('请输入key');
    console.log(obj[k]);
}

# 3.说一下你对原型链的理解、应用场景，以及下面代码报错原因。

function Dog(name) {  
	this.name = name
}

Dog.bark = function() {
	console.log(this.name + 'says  woof')
}	

let fido = new Dog('fido')；
fido.bark()

理解：在javaScript 中我们是使用构造函数来创建一个对象的，每个构造函数的内部都有一个prototype属性，指向它的原型对象。构造函数的对象可以通过一个不规范的 proto 属性或者ES5新增的Object.getPrototypeOf() 方法来访问这个原型对象。当我们访问一个对象的属性时，如果这个对象内部不存在这个属性，那么它就会去它的原型对象里找这个属性，这个原型对象又会有自己的原型，于是就这样一直找下去，直到为null，这就是原型链。

应用场景：js通过原型链实现函数或对象的继承

报错原因：fido这个对象并没有bark方法。代码中虽然给dog对象绑定了bark方法，但是绑定的是静态方法，没有绑定到原型上，调用形式只能通过dog.bark去调用。Fido并没有继承这个方法，所以会报错。

如果想要不报做，需要在dog函数的内部写 bark 方法，或者使用 Dog.prototype.bark = function() {}

# 4. 事件委托是什么？有什么作用以及原理。

事件委托就是子元素不处理事件，把事件委托给父元素去处理。

作用：

1、减少页面绑定事件的数量，提高性能

2、后添加的子节点，无需重新绑定事件

原理：通过事件冒泡。把事件加到父级上，触发事件执行效果。

# 5.下面代码执行结果是什么，为什么？

function greet(person) {
  if (person == { name: 'amy' }) {
   	return 'hey amy'
  } else {
    return 'hey arnold'
  }
}

greet({ name: 'amy'})

执行结果是 'hey arnold'。

原因：greet方法传入的为对象，对象在栈中是赋值为一个堆中的引用地址，对象实际存储在堆中。

利用 == 判断，地址不同，返回 false，执行 else 中的语句。

# 6.写出下面代码执行后正确顺序的输出结果以及原因解析。

var a=1;
function fun1() {
	console.log(a);
}
function fun2() {
	console.log(a);
	var a=2;
	console.log(a);
}
fun1();
fun2();

console.log(a);

结果：1 undefined 2 1

解析：首先执行 fun1 (), 打印a 没找到，在全局中找 a = 1，输出 1 。

然后执行 fun2(), 函数内部 var 存在变量提升，var a = underfined, 输出 underfined，

然后赋值 a = 2, 输出 2，

最后全局输出的为全局变量 a 为 1

# 7.写出下面代码执行后正确顺序的输出结果以及原因解析。

var a=1;
console.log(1);
setTimeout(function () {
  console.log(2);
},0);

function fun() {
  console.log(3);
  setTimeout(function () {
    console.log(4);
  },0);
  for (var i=0;i<10000;i++){
    a++;
  }
  console.log(5);
  return a;
};

console.log(6);
console.log(fun());

输出结果：1 6 3 5 10001 2 4

解析：JS 解析代码，从上到下，遇 var 先声明，遇 函数方法声明加定义。

首先 输出 1。

然后遇到第一个宏任务 setTimeout，放入异步队列，等待主线程执行完再执行。然后输出 6 。

执行fun函数， 输出3，第二个 setTimeout 放异步队列。执行for 循环，a 最终值为 10001。输入 5，返回a, 输出a 的值为 10001。

主线完成，队列是先进先出结构，添加第一个 setTimeout 到主线程，执行输出 2，在执行第二个 setTimeout 输出 4。
